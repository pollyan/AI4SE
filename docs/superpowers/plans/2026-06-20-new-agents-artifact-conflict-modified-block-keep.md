# New Agents Artifact 冲突修改块保留服务端计划

## 范围

补齐 Artifact 保存冲突中修改块的反向决策能力，让用户可以一键保留服务端修改块。

## 步骤

1. 补失败测试：
   - 构造服务端较新版本和用户草稿同位置多行修改。
   - 打开冲突 diff。
   - 期望存在 `保留服务端修改块` 按钮。
   - 点击后编辑草稿等于服务端当前版本，并写入审计事件。
2. 实现审计事件：
   - 扩展修改块审计函数支持 `artifact_merge_block_modified_kept`。
   - 文案区分 `采纳草稿修改块` 和 `保留服务端修改块`。
3. 实现按钮行为：
   - 复用已有修改块识别结果。
   - 设置 `editDraft` 为服务端内容。
   - 写入活动轨迹。
4. 验证：
   - 聚焦测试先失败后通过。
   - 运行 ArtifactPane 与 store 相关测试。
   - 运行 frontend lint/build 与 `git diff --check`。
