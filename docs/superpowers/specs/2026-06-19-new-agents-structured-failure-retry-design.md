# New Agents 结构化失败重试体验设计

## 背景

当 Agent Runtime 返回 `SCHEMA_VALIDATION_FAILED` 或超过结构化输出重试次数时，`chatService` 已经会生成友好文案，并说明右侧产出物保持不变、可以重试。但 ChatPane 当前只把这段文本作为普通 Markdown 渲染，真正的 `重试` 按钮藏在消息底部的次级操作区。用户看到“请重试”时，很难立刻发现可执行入口。

## 目标

- 结构化输出失败的 assistant 消息在左侧渲染为轻量恢复卡片。
- 卡片清楚说明：本轮没有更新有效结构化结果，右侧产出物保持不变。
- 卡片提供主操作 `重试本阶段生成`，复用现有 `handleRetry` 与 artifact rollback。
- 如果结构化失败消息和阶段推进确认同时存在，隐藏阶段推进确认，避免用户带着无效产物继续推进。

## 非目标

- 不新增重型 chat schema，不把自然聊天强制改造成固定字段。
- 不修改 Agent Runtime、typed SSE、artifact contract 或后端接口。
- 不在本切片实现 Mermaid 图表块的重新生成策略。
- 不在本切片实现供应商错误与设置/运行统计的联动。

## 用户体验

用户看到结构化失败时，消息气泡内会出现一个醒目的恢复区。恢复区包含失败标题、产物状态说明、连续失败建议，以及一个主按钮 `重试本阶段生成`。用户点击后，系统回到上一轮用户输入草稿并恢复右侧产物状态，沿用已有 retry/rollback 逻辑。

如果当前 store 中仍有阶段推进确认，但最新 assistant 消息是结构化失败，ChatPane 不展示 `确认进入下一阶段`，避免错误引导。

## 验收

- ChatPane 测试证明结构化失败消息展示恢复卡片和主重试按钮。
- ChatPane 测试证明点击 `重试本阶段生成` 调用现有 `handleRetry`。
- ChatPane 测试证明结构化失败时不展示阶段推进确认。
- chatService 既有测试继续证明失败前右侧产物保持不变、retry/rollback 可用。
