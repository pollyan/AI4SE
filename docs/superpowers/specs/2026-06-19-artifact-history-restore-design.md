# Artifact 历史版本恢复设计

## 背景

`docs/todos/new-agents-evolution.md` #11 已经完成历史版本 diff 第一片，用户可以在 Artifact 历史弹层中查看旧版本并与当前产出物做行级对比。但当前只能看，不能把一个可信旧版本恢复成当前工作区内容。对于模型生成结果变差、误改或用户想回退方案时，这条协作链路仍不闭环。

## 用户故事

作为 New Agents 用户，当我在右侧产出物历史版本中发现某个旧版本更合适时，我希望点击“恢复此版本”，让当前产出物回到该版本，同时保留恢复前的当前内容为一个新的历史版本，方便再次回退。

## 范围

进入本轮：

- 在 Artifact 历史版本弹层中新增“恢复此版本”操作。
- 恢复时先把当前 `artifactContent` 作为当前 stage 的新历史版本保存到本地 `artifactHistory`。
- 再通过现有 `setArtifactContent()` 把当前产出物切到所选历史版本内容，并关闭历史弹层。
- 补充组件测试覆盖恢复行为和恢复前内容保留。

不进入本轮：

- 不新增服务端 artifact update API。
- 不做逐行接受/拒绝。
- 不做权限、多人协作锁或批注。
- 不改变历史版本列表的数据模型。

## 验收条件

1. 有历史版本时，历史弹层显示“恢复此版本”。
2. 点击后当前产出物内容变为选中历史版本内容。
3. 恢复前的当前内容会追加到 `artifactHistory`，stageId 为当前 stage。
4. 无选中历史版本时不显示恢复操作。
5. 相关 ArtifactPane 测试、TypeScript 检查和 `git diff --check` 通过。
