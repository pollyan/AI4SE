# New Agents 工作区操作降噪设计

## 背景

Lisa / Alex 工作区 Header 当前把新会话、历史会话、上下文摘要、运行统计、测试资产、导出报告、设置并列展示。用户在阶段工作流中看到过多等权重操作，容易误判哪些是主链路动作，哪些是辅助治理或产物动作。

右侧 ArtifactPane 已经提供 Markdown / Word / PDF 下载入口，Header 中的导出报告形成重复归属。ArtifactPane 生成态还包含一条没有真实进度来源的横向进度条，用户会把它理解成卡住或无意义的进度。

## 目标

- Header 一级操作只保留高频工作区动作：新会话、历史会话、更多操作。
- 上下文摘要、运行统计、测试资产、设置进入“更多操作”菜单。
- 产物导出只保留在 ArtifactPane 工具条，Header 不再提供导出报告按钮。
- ArtifactPane 生成态保留明确文案和轻量动画，移除无法表达真实进度的横向进度条。

## 非目标

- 不改变共享 Agent Runtime、SSE、workflow manifest 或后端接口。
- 不在本切片实现结构化输出失败重试卡片。
- 不调整阶段条的跳转权限和状态语义。
- 不迁移测试资产到独立右侧资产上下文；本切片先从 Header 一级操作中收敛。

## 用户体验

用户进入工作区后，顶部只看到当前 workflow / 阶段、创建新会话、查看历史会话和更多操作。需要诊断、治理或配置时，从“更多操作”进入对应能力。需要导出产出物时，在右侧产物工具条下载，符合产物操作归属。

生成右侧产物时，用户看到“正在构建右侧产出物”和非确定性加载动画，不再看到无法与真实进度对应的横向进度条。

## 验收

- Header 测试证明导出报告不再作为 Header 按钮出现。
- Header 测试证明更多菜单可打开上下文摘要、运行统计和 TEST_DESIGN 测试资产。
- ArtifactPane 测试证明生成态不再渲染装饰性横向进度条，同时保留生成态文案和动画。
