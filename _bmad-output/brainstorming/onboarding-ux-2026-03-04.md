## Technique Execution

### Phase 1: Role Playing — 三种用户首次进入的心理画像

#### 👤 角色 A：小白测试员"小李"（入职 3 个月）
- 被 Lead 甩了链接说"用这个工具"
- 对 AI 工具没概念，不确定"测试设计"是什么
- **核心心理：** 紧张 + 茫然 + 害怕问出蠢问题
- **内心独白：** "输入需求？啥需求？我连需求文档都没看完......"

#### 👤 角色 B：资深测试工程师"老王"（8 年经验）
- 自己来试试，习惯用 ChatGPT
- 知道自己要什么，但不知道工具的"套路"
- **核心心理：** 效率导向 + 想快速上手

#### 👤 角色 C：QA Lead"张姐"（管理 5 人团队）
- 评估工具是否值得推广
- 关注能力边界和专业度
- **核心心理：** 评估者视角 + 需要快速判断价值

#### Phase 1 核心发现

> **不管是小白、老手还是管理者，进入空白聊天页面时的核心痛点是一致的——"我不知道该说什么来开始"。**
>
> 解决方案需要同时覆盖三个层面，且可以用统一方案处理：
> - 🛡️ **心理安全感**："别怕，我来带你"
> - 💡 **能力展示**："我能帮你做这些"
> - 🎯 **行动指引**："你现在点这个就行"

### Phase 3: SCAMPER — 对现有空状态做系统化改造

**当前基准（ChatPane.tsx 第 61-68 行）：**
一个灰色 Bot 图标 + "你好，我是 Lisa。请在下方输入您的需求..." 的单行静态文案

| 镜头 | 改造动作 |
|------|---------|
| **S (替换)** | 替换死板单行文案 → 结构化分段欢迎语；替换被动 placeholder → 动态引导性 placeholder |
| **C (组合)** | 欢迎文案 + 可点击示例卡片 组合在一起；示例卡片内容 由工作流配置驱动 |
| **A (借鉴)** | 借鉴 ChatGPT 的卡片布局；借鉴 Copilot 的"AI 先说话"心理暗示 |
| **M (放大/缩小)** | 放大专业度展示；缩小信息量（示例最多 3 个） |
| **P (另作他用)** | 示例数据未来复用到工作流卡片页 |
| **E (去掉)** | 去掉巨大灰色 Bot 图标；去掉"请在下方输入"的废话 |
| **R (反转)** | 反转主被动关系：AI "主动"展示在等用户 |

## Selected Ideas

### ✅ 最终方案：Onboarding Welcome Kit（冷启动欢迎套件）

#### 方案概述

当用户进入一个工作流的聊天页面且对话历史为空时，在聊天区域中央展示一个"欢迎套件"，包含三个核心元素。用户发出第一条消息后，套件自动消失，进入正常对话流。

#### 核心组成

| 元素 | 描述 | 覆盖的需求层 |
|------|------|------------|
| **1. 欢迎语块** | 角色介绍 + 本工作流能力概述 + 当前阶段说明 | 🛡️ 心理安全感 + 💡 能力展示 |
| **2. 示例提问卡片** | 2-3 个可点击卡片，点击后**直接发送**（零门槛触发） | 🎯 行动指引 |
| **3. 动态 Placeholder** | 输入框 placeholder 跟随当前工作流变化 | 🎯 行动指引（微引导） |

#### 交互规则

- 展示条件：`chatHistory.length === 0`
- 消失条件：用户发出第一条消息
- 卡片点击行为：**直接发送**（不填入输入框，直接触发 `handleSend`）
- 实现方式：**空状态 UI 组件**（不进入对话历史，不改动 store）

#### 数据配置位置

在 `workflows.ts` 的 `WorkflowDef` 类型中扩展以下字段：

```typescript
interface WorkflowDef {
    id: string;
    name: string;
    stages: StageDef[];
    // === 新增: Onboarding Welcome Kit ===
    onboarding: {
        welcomeMessage: string;      // 欢迎语（支持 Markdown）
        starterPrompts: string[];    // 2-3 个示例提问
        inputPlaceholder: string;    // 输入框 placeholder
    };
}
```
